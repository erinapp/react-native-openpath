OpenpathMobileAccessCoreDelegate

public protocol OpenpathMobileAccessCoreDelegate : AnyObject
  • Delegate class for receiving asynchronous results and events from OpenpathMobileAccessCore *
  • Create a class that conforms to this protocol and assign it to delegate. *
  • On successful callbacks, the second parameter in each function is a Dictionary called message with a root key of “data”.
  • The value of the root entry varies and is documented for each function. *
  • If the callback is reporting an error condition, message has the following shape:
  • [ "err": [ "message": String, "code": String ] ]

  • openpathMobileAccessCore(_:onUnprovisionResponse:)

    Invoked when a user is unprovisioned.

    The unprovision may be triggered by a call to unprovision(userOpal:) in the SDK, but also if the SDK detects that the active user is not allowed to be provisioned, for example if the same user using the same credential provisioned the SDK on another device.

    The reasonCode and the reasonDescription objects returned in the message payload clarify the reason for this unprovision.

  • openpathMobileAccessCore(_:onItemsSet:)

    Invoked when item information is set. Contains the full set of items and fields.

    This is the event that lists the entries the user has access to. It is called any time user access to entries changes (e.g., through updates in the portal).

    • items is a dictionary where the keys are entry-{entryId} or reader-{readerId}.

    • itemOrdering is an array that represents the ordering and hierarchy of items. Typically, in a non-elevator scenario, you will only see entry itemTypes and no subItems. In an elevator scenario however, you will see reader itemTypes (which normally represents the elevator), and entry subItems (which normally represents the floors that are tied to that reader).

    Success message::

    [
  "data": [
    "items": [
      "entry-{entryId}": [
          "acuId": Int,
          "cameraIds": [Int],
          "color": String,
          "isAvailableInRange": Bool,
          "isAvailableNearbyNotification": Bool,
          "isAvailableOverrideUnlock": Bool,
          "isInRange": Bool,
          "itemId": Int,
          "itemType": String,  // "entry"
          "name": String,
          "orgName": String,
          "readerIds": [Int],
        ],
        "reader-{readerId}": [
        "acuId": Int,
        "cameraIds": [Int],
        "color": String,
        "isAvailableInRange": Bool,
        "isAvailableNearbyNotification": Bool,
        "isAvailableOverrideUnlock": Bool,
        "isInRange": Bool,
        "itemId": Int,
        "itemType": String,  // "reader"
        "name": String,
        "orgName": String,
        "readerIds": [Int],
        ],
    ],
    "itemOrdering": [[
      "itemId": Int,
      "itemType": String, // "entry" or "reader"
      "subItems": [[String:Any]]
    ]]
  ]
]

  • openpathMobileAccessCore(_:onItemsUpdated:)

    Invoked when item information is updated. Only contains items and fields that have been updated.

    This is called for any items whose information has changed (e.g., whether it became in or out of range, whether the schedule has changed, etc.).

    This is for changes to existing items only. If new items have been discovered or any items have been removed since the last update, this is not called, but onItemsSet is called instead.

    Success message::

    [
  "data": [
    "items": [
      "entry-{entryId}": [
        "acuId": Int,
        "cameraIds": [Int],
        "color": String,
        "isAvailableInRange": Bool,
        "isAvailableNearbyNotification": Bool,
        "isAvailableOverrideUnlock": Bool,
        "isInRange": Bool,
        "itemId": Int,
        "itemType": String,  // "entry" or "reader"
