Release v0.2.0

Release Notes

  • Feature: added getErrors function to get all error codes and descriptions
  • Feature: added requestAuthorization function to request permissions for “location”, “motion”, “bluetooth” and “notification”.
  • Feature: added getAuthorizationStatuses function to request permission authorization statuses
  • Feature: added getReadersInRange function to get Openpath readers in range
  • Feature: added getUserApiToken function to retrieve logged in user’s Openpath API token
  • Feature: added sendFeedback function (and onSendFeedbackResponse delegate) to send feedback to Openpath. This feature should be used for development, testing and integration purposes only
  • Feature: added onMotionStatusChanged delegate to report motion status changed
  • Feature: added enableErrorNotificationsForItem to enable/disable error notifications for an entry or reader.
  • Improvement: all unlock responses (now including Wave) will be reported via a new delegate function onUnlockResponse
  • Improvement: our SDK and its dependencies are now distributed using binary frameworks so that the same frameworks can be used across simulator, device, and different Swift & XCode versions. This change will also remove the need to use CocoaPods for our SDK dependencies.
  • Improvement: beacon region monitoring reduced from 20 to 10 regions so that 10 are free for your use (Apple allows a max limit of 20 regions per app)

Migration Guide

Use binary frameworks

Adding the framework

Remove the old version of our SDK from your project. Copy the frameworks directory (provided with our new SDK) to your project. From your Xcode project root, go to the General tab, under Embedded Libraries, click on the + button. Browse to the frameworks directory (the one that we just added) and select the entire list of frameworks, then verify that under the column Embed all the frameworks are set as Embed & Sing.

Upgrade to New Build System

If you were using Legacy Build System, you need to upgrade to New Build System because the legacy system doesn’t support Binary Frameworks. Please go to File-> Workspace Settings (or Project Settings if you’re not in a workspace), and select New Build System for both Shared Workspace & Per-User Workspace Settings.

Remove CocoaPods

If you were using the CocoaPods Engine just for the purpose of installing dependencies for our SDK, here is a guide to remove it from you project:

sudo gem install cocoapods-deintegrate cocoapods-clean
pod deintegrate
pod cache clean --all
rm Podfile

NOTE: Thanks to Haiyesbye for his post on Medium.

Unlock function

The unlock function’s completion callback was removed so the function now looks like this:

func unlock(itemType: String, itemId: Int, requestId: Int, timeout: Int)

The unlock response is now handled via a new delegate function:

func openpathMobileAccessCore(_ openpathMobileAccessCore: OpenpathMobileAccessCore, onUnlockResponse message: [String: Any])

The message payload returned in the delegate function remains the same and did not change from the message that was in the completion callback.

The purpose of this change is so that all unlock responses, including Wave, can be received. Prior to this change, only responses for unlocks that were called via the unlock function would be received.
