Release v0.1.0

Release Notes

  • Feature: added support for provisioning multiple user accounts and switching between them
  • Feature: added support for the “geofence” location-based unlock method
  • Feature: added “getSdkVersion” method
  • Improvement: included error descriptions in unlock responses
  • Improvement: improved error descriptions when unlock fails
  • Improvement: wave/touch reliability improved upon app restart or device reboot

Bug

Bug fix: only remove keychain and storage items (UserDefaults) created within SDK instead of everything created within app

Migration Guide

OpenpathMobileAccessCore was updated to optimize the process of user initialization and to provide further support for applications with multiple user’s accounts.

The following function is deprecated:

func start(completion: @escaping (_ message: [String: Any]) -> Void)

The following functions are introduced:

func switchUser(userOpal:String, completion: @escaping (_ message: [String: Any]) -> Void)

func syncUser(completion: @escaping (_ message: [String: Any]) -> Void)

Provision flow

If you want to provision a new user, the code should be updated from calling provision and then start to the following 3-step process.

  1. call provision to provision a user
  2. call switchUser to activate the user
  3. call syncUser to receive all updated information for that user

Switch User flow

If you simply want to switch the current user, you will use the following 2-step process.

  1. call switchUser to activate the user
  2. call syncUser to receive all updated information for that user

Initializitation flow

When the app first opens but is already provisioned, the code should be updated from just calling start to the following process.

  1. call syncUser to receive all updated information for that user
